# Visual Analysis Protocol

Inspect the visual before reading the student's answer in detail. This is mandatory: do not score, rewrite, or generate a model answer from the student's text alone.

## Universal Checklist

- Identify visual type: bar chart, line graph, table, pie chart, map, process, mixed chart.
- Read the task instruction and visual title.
- Record units: percentage, number, money, age, years, stages, locations.
- Identify categories, legends, axes, labels, and time periods.
- Identify the main features before details.
- Check whether exact values are printed or must be estimated.
- Do not invent precise values when the chart only supports approximation.

## Bar Charts

- Identify categories on each axis and legend groups.
- Find highest and lowest bars.
- Compare major groups rather than list every bar.
- Use `approximately`, `around`, `just under`, `slightly above`, and `roughly`.
- Watch for percentage/proportion wording.

## Line Graphs

- Identify starting and ending values, peaks, troughs, and major trend changes.
- Group lines by similar trends where possible.
- Avoid describing every small movement.
- Use trend language: rose, declined, remained stable, fluctuated, peaked, reached a low.

## Tables

- Identify row and column categories.
- Select key highest/lowest values, major contrasts, and exceptions.
- Avoid turning the table into a list.
- Group data by meaningful similarities.

## Pie Charts

- Identify each category and percentage/share.
- Compare largest and smallest shares.
- If there are multiple pies, compare changes across time or groups.

## Maps

- Identify before/after or location differences.
- Describe major changes in layout, land use, facilities, roads, and orientation.
- Use location language: to the north, in the centre, adjacent to, replaced by.

## Process Diagrams

- Identify start and end points.
- Count stages.
- Group stages if the process is long.
- Use passive voice where natural.
- Do not add causes or opinions.

## Mixed Charts

- Identify what each visual contributes.
- Summarize the relationship between visuals.
- Avoid overloading the answer with every value.

## Internal Visual Facts Note

Before reviewing, create a compact internal note:

```text
Type:
Units:
Categories/time:
Main overview:
Key comparisons:
Potential student traps:
```

Keep this note internal by default. Use it to drive comments, rewrites, scoring, and the model answer; only show it in the reviewed DOCX if the user asks for it.

## Sample Similarity Selection

After identifying the visual type, consult `teacher_samples_index.md` and pick the closest teacher-reviewed samples:

- Same visual type first: bar chart, line graph, table, pie chart, map, process, or mixed visual.
- Same reporting challenge second: trend comparison, proportions, process sequencing, before/after map layout, or mixed data grouping.
- Prefer one or two close samples over loading many weakly related samples.
